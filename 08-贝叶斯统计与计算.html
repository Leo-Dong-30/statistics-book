贝叶斯统计与计算

频率学派问：“如果参数固定，数据出现的概率是多少？”贝叶斯学派问：“给定数据，参数取某个值的信念应该多少？”两个问题看似对称，却通向截然不同的统计哲学。本章以前半部分铺设理论地基，后半部分建造计算引擎——两者合一，才是完整的贝叶斯工具箱。

贝叶斯范式

频率学派 vs 贝叶斯学派

前面几章（Ch05-07）介绍的参数估计和假设检验，本质上属于频率学派（Frequentist）的框架。在频率学派看来：

  • 参数 \(\theta\) 是固定的、未知的常数。
  • 数据是随机的，来自 \(P(X \mid \theta)\)。
  • 推断基于重复抽样的性质：无偏性、相合性、显著性水平等。

贝叶斯学派（Bayesian）则采取了完全不同的立场：

  • 参数 \(\theta\) 本身也是随机的——它服从一个先验分布 \(P(\theta)\)。
  • 数据一旦被观测到，就是固定的。
  • 推断基于给定数据下参数的条件分布 \(P(\theta \mid \text{data})\)，即后验分布。

这两种范式的差异，根源在于对”概率是什么”这一元问题的不同回答。频率学派认为概率是长期频率的极限；贝叶斯学派则认为概率是信念（或知识状态）的量化。

贝叶斯定理回顾

贝叶斯定理在 Ch01 中已经出现。将其应用于统计推断：

\[p(\theta \mid x) = \frac{p(x \mid \theta) \, p(\theta)}{p(x)} = \frac{p(x \mid \theta) \, p(\theta)}{\int p(x \mid \theta) \, p(\theta) \, d\theta}\]

其中：

  • \(p(\theta)\)：先验分布——在观测数据之前对 \(\theta\) 的信念。
  • \(p(x \mid \theta)\)：似然函数——在参数 \(\theta\) 下观测到数据 \(x\) 的概率。
  • \(p(\theta \mid x)\)：后验分布——在观测数据之后对 \(\theta\) 的更新信念。
  • \(p(x)\)：边际似然（或证据）——归一化常数，确保后验是合法的概率分布。

由于 \(p(x)\) 不依赖于 \(\theta\)，后验分布常常写为比例形式：

\[p(\theta \mid x) \propto p(x \mid \theta) \, p(\theta)\]

后验 ∝ 似然 × 先验。这就是贝叶斯学习的核心公式。

贝叶斯更新：学习的形式化

贝叶斯定理给出了一个精确的”学习”模型：

\[P(\theta \mid \text{新数据}) \propto P(\text{新数据} \mid \theta) \cdot P(\theta)\]

其中 \(P(\theta)\) 是你在看到新数据之前的信念（先验）。关键递归：今天的后验 = 明天的先验。当你看到第二批数据时，昨天的后验自动成为今天的先验，贝叶斯更新不需要区分”旧数据”和”新数据”——它天然支持增量学习。

频率学派问：“如果参数固定，这批数据出现的概率是多少？”贝叶斯学派问：“给定这批数据，我对参数的信念应该更新为多少？”前者是数据的概率，后者是信念的修正。两派的核心分歧，在于”概率”本身被视为客观频率还是主观信念。但无论你站在哪边，贝叶斯更新作为信息处理的数学模型，其优雅性无可争议。

先验分布的选择

共轭先验

如果先验分布与后验分布属于同一分布族，则称该先验为共轭先验（Conjugate Prior）。共轭先验大大简化了后验的计算——观测者只需更新分布参数，无需重新推导整个后验形式。

似然（数据分布） 参数 共轭先验 后验
二项 \(B(n, \theta)\) \(\theta\) Beta\((\alpha, \beta)\) Beta\((\alpha + x, \beta + n - x)\)
泊松 \(P(\lambda)\) \(\lambda\) Gamma\((\alpha, \beta)\) Gamma\((\alpha + \sum x_i, \beta + n)\)
正态（均值，方差已知） \(\mu\) Normal\((\mu_0, \tau_0^2)\) Normal（加权平均）
正态（方差，均值已知） \(\sigma^2\) Inv-Gamma\((\alpha, \beta)\) Inv-Gamma（更新参数）
多项分布 \(\mathbf{p}\) Dirichlet\((\boldsymbol{\alpha})\) Dirichlet\((\boldsymbol{\alpha} + \mathbf{n})\)

例子：硬币的公平性

设抛一枚硬币 \(n = 20\) 次，观察到 \(x = 13\) 次正面。设先验为 \(\theta \sim \text{Beta}(2, 2)\)（相当于事先”看过”2 正 2 反）。

后验：\(\theta \mid x \sim \text{Beta}(2 + 13, 2 + 7) = \text{Beta}(15, 9)\)

后验均值：\(E[\theta \mid x] = \frac{15}{15+9} = 0.625\)。而 MLE 为 \(\hat{\theta} = 13/20 = 0.65\)。先验的”收缩”效应将后验均值拉向先验均值 \(0.5\)。

Code 
n <- 20; x <- 13
alpha_prior <- 2; beta_prior <- 2
alpha_post <- alpha_prior + x; beta_post <- beta_prior + n - x

theta <- seq(0, 1, length.out = 200)
plot(theta, dbeta(theta, alpha_post, beta_post), type = "l", lwd = 2,
     xlab = expression(theta), ylab = "密度", main = "先验、似然与后验")
lines(theta, dbeta(theta, alpha_prior, beta_prior), lty = 2, col = "blue")
lines(theta, dbeta(theta, x + 1, n - x + 1), lty = 3, col = "red")
legend("topright", c("后验", "先验 Beta(2,2)", "似然"), lty = 1:3, col = c(1, "blue", "red"))
abline(v = alpha_post / (alpha_post + beta_post), lty = 4)

无信息先验

当我们缺乏先验知识时，可以使用无信息先验（Non-informative Prior），让数据”自己说话”。常见的无信息先验包括：

  • Laplace 先验（均匀分布）：\(p(\theta) \propto 1\)。适用于参数空间有界的情况。
  • Jeffreys 先验：\(p(\theta) \propto \sqrt{I(\theta)}\)，其中 \(I(\theta)\) 为 Fisher 信息量。Jeffreys 先验在一对一变换下保持不变，具有良好的理论性质。
  • 位置参数的均匀先验：\(p(\mu) \propto 1\)。
  • 尺度参数的倒数先验：\(p(\sigma) \propto 1/\sigma\)。

注意：无信息先验可能是非正常（improper）的——即积分不为 1。但只要后验是正常的，就不会出问题。

层次先验

当先验本身含有未知参数时，可以对这些超参数再设定先验，形成层次模型（Hierarchical Model）：

\[p(\theta, \phi \mid x) \propto p(x \mid \theta) \, p(\theta \mid \phi) \, p(\phi)\]

层次模型自然地处理了”不同组的参数之间存在相似性”的问题，在随机效应模型、多臂老虎机问题、贝叶斯优化等场景中应用广泛。

贝叶斯推断

点估计

后验分布是一个完整的分布，而不是一个单点。但如果需要点估计，常用的选择是：

  • 后验均值：\(\hat{\theta}_{\text{mean}} = E[\theta \mid x]\)——最小化平方损失下的最优估计。
  • 后验中位数：\(\hat{\theta}_{\text{med}}\)——最小化绝对损失下的最优估计。
  • 最大后验估计（MAP）：\(\hat{\theta}_{\text{MAP}} = \arg\max_\theta p(\theta \mid x)\)——最大化后验密度。

MAP 估计与 MLE 的关系：由于 \(p(\theta \mid x) \propto p(x \mid \theta) p(\theta)\)，有

\[\log p(\theta \mid x) = \log p(x \mid \theta) + \log p(\theta) + \text{const}\]

因此，MAP 等价于在似然函数上加了一个来自先验的正则化项。这是连接频率学派与贝叶斯学派的重要桥梁——Ch06 提到的”用偏差换方差”在 MAP 估计中体现为”用先验收缩换取稳定性”。

区间估计：可信区间

贝叶斯的区间估计是可信区间（Credible Interval），与频率学派的置信区间有本质区别：

  • 置信区间：\(P(\text{区间包含} \, \theta) = 0.95\)——这个概率是对”区间”的随机性而言的（重复抽样下 95% 的置信区间包含 \(\theta\)）。
  • 可信区间：\(P(\theta \in \text{区间} \mid \text{data}) = 0.95\)——这个概率直接是对”参数”的信念而言的（给定数据，\(\theta\) 有 95% 的概率落在该区间内）。

贝叶斯可以直接说：“我有 95% 的把握认为参数在 \([a, b]\) 之间。”频率学派则不能说这句话——这是两种范式最直观的差异。

常用的可信区间： - 等尾可信区间：取后验的 \(\alpha/2\) 和 \(1-\alpha/2\) 分位数。 - 最高后验密度（HPD）区间：在给定概率下，取后验密度最高的区域。HPD 区间是最短的可信区间。

假设检验：贝叶斯因子

贝叶斯假设检验不是通过 \(p\) 值，而是通过比较两个模型的后验概率比来进行：

\[\frac{P(H_1 \mid x)}{P(H_0 \mid x)} = \frac{P(x \mid H_1)}{P(x \mid H_0)} \cdot \frac{P(H_1)}{P(H_0)}\]

其中，\(BF_{10} = \frac{P(x \mid H_1)}{P(x \mid H_0)} = \frac{\int p(x \mid \theta_1) p(\theta_1) d\theta_1}{\int p(x \mid \theta_0) p(\theta_0) d\theta_0}\) 称为贝叶斯因子（Bayes Factor）。

\(\log_{10} BF_{10}\) \(BF_{10}\) 证据强度
\(< 0\) \(< 1\) 支持 \(H_0\)
\(0\) — \(0.5\) \(1\) — \(3.2\) 弱
\(0.5\) — \(1\) \(3.2\) — \(10\) 中等
\(1\) — \(2\) \(10\) — \(100\) 强
\(> 2\) \(> 100\) 决定性

贝叶斯因子自动”惩罚”模型复杂度——复杂模型的先验预测分布更分散（它”预期”了更多种可能的数据），因此需要更强的数据支持才能获得大的贝叶斯因子。这体现了奥卡姆剃刀原则：如无必要，勿增实体。

这一自动惩罚机制在信息论上对应的是：复杂模型需要更多的信息（更极端的证据）来证明自己的”必要性”。如果你的数据只是稍微偏离简单模型的预测，贝叶斯因子会判断这是抽样误差而非真实效应——从而保护你不被噪声诱惑去选择不必要的复杂模型。

贝叶斯预测

贝叶斯方法自然地提供了预测分布。给定观测数据 \(x\)，新观测 \(x^*\) 的预测分布为：

\[p(x^* \mid x) = \int p(x^* \mid \theta) \, p(\theta \mid x) \, d\theta\]

这被称为后验预测分布（Posterior Predictive Distribution）。它不是用一个点估计 \(\hat{\theta}\) 代入 \(p(x^* \mid \hat{\theta})\)，而是对参数的不确定性做积分平均。这一区别意味着贝叶斯预测自然地包含了参数不确定性，通常比基于点估计的预测更稳健。

贝叶斯统计的优缺点

优点： 1. 推断解释直觉自然（“参数有 95% 概率落在区间内”）。 2. 自然地融入先验知识。 3. 自动的正则化——先验防止过拟合。 4. 预测分布自然地包含了参数不确定性。 5. 在小样本下表现通常优于频率学派方法。

缺点： 1. 先验的选择可能主观，不同先验可能导致不同结论（敏感性分析是必要的）。 2. 计算复杂——后验往往没有解析形式，需要 MCMC 等数值方法。 3. 高维问题中的计算挑战。

所谓的真理，不过是不断修正的偏见。 先验是你的偏见——它来自于你已有的知识和经验。数据是修正力——它用客观证据不断打磨你的主观判断。贝叶斯统计不掩饰主观性，而是将它纳入形式化的框架：你明确写下你的偏见，然后用数据公开地修正它。

贝叶斯计算：当后验没有解析解

前面五节，观测者始终假设后验可以解析地写出——要么有共轭先验担保闭式，要么分布形式足够简单。但真实的统计建模远不总是如此。高维参数空间、非共轭先验、复杂的似然结构，都让后验 \(p(\theta \mid x)\) 变成一个”写得出比例式但算不出归一化常数”的数学对象。

观测者面临的核心挑战是：后验固定了（给定先验和数据），但无法直接求解它。 贝叶斯分析的本质工作——无论是点估计、区间估计还是预测——都归结为对后验分布做摘要（求期望、求分位数、求积分）。当闭式不存在时，观测者需要另一套武器：不是”推导”，而是”采样”。

计算策略按近似精度与通用性递增排列：

  • 高斯近似：用正态分布局部逼近后验峰——快速但有偏。
  • 独立蒙特卡洛：从提议分布采样，用拒绝或加权修正——在低维有效。
  • MCMC：构造马尔可夫链使其平稳分布恰为目标后验——高维问题的通用解。

高斯近似

当样本量足够大时，后验分布在 MAP 估计附近近似为正态分布。这就是贝叶斯中心极限定理：后验分布随样本量增大趋向于以 MAP 为中心、以 Fisher 信息量的逆为方差的正态分布。

\[\theta \mid Y \sim \text{Normal}\left[\hat{\theta}_{\text{MAP}},\; I(\hat{\theta}_{\text{MAP}})^{-1}\right]\]

其中 \(I_{jk} = -\frac{\partial^2}{\partial\theta_j\partial\theta_k} \log p(\theta \mid Y)\) 是后验对数密度的负二阶导（观测信息矩阵）。

这一近似的直觉来自 Ch06 的 MLE 渐近正态性：当 \(n\) 很大时，先验的影响被似然碾压，后验的形状由似然函数在峰值处的曲率决定。优势在于计算极快——只需找到 MAP 点（数值优化一次）并计算该点的 Hessian 矩阵。代价是忽略了后验可能的多峰、厚尾或偏斜。

例：二项试验的胜率估计。一支队伍上赛季 38 场比赛赢了 30 场，观测者希望估计其真实胜率 \(\theta\)。

MLE 直接求解似然函数 \(L(\theta) = C_{38}^{30} \theta^{30} (1-\theta)^8\)，导得 \(\hat{\theta}_{\text{MLE}} = 30/38 \approx 0.789\)。当样本充足时没有问题。但如果赛季刚开始、只打了 5 场赢了 2 场呢？MLE 给出 \(0.4\)，但观测者此前对这支队伍有了解——历史胜率约 50%。MAP 用 Beta 先验将这一外部知识注入：

先验：\(\theta \sim \text{Beta}(10, 10)\)（相当于事先”看过”10 胜 10 负，集中在 0.5 附近）

后验：\(p(\theta \mid \text{data}) \propto \theta^{30}(1-\theta)^8 \cdot \theta^{9}(1-\theta)^{9} = \theta^{39}(1-\theta)^{17}\)

\[\hat{\theta}_{\text{MAP}} = \frac{k + \alpha - 1}{n + \alpha + \beta - 2} = \frac{30 + 10 - 1}{38 + 10 + 10 - 2} = \frac{39}{56} \approx 0.696\]

先验将估计从 MLE 的 0.789 拉回到 0.696——这就是先验的”收缩”效应。

例：设 \(Y \mid \theta \sim B(n, \theta)\)，\(\theta \sim \text{Beta}(0.5, 0.5)\)，求 \(p(\theta \mid Y)\) 的高斯近似。

后验：\(\theta \mid Y \sim \text{Beta}(Y + 1/2,\; n - Y + 1/2)\)。令 \(A = Y - 0.5\)，\(B = n - Y - 0.5\)，则 MAP 估计 \(\hat{\theta} = A / (A + B)\)，后验方差近似为 \(\left(\frac{A}{\hat{\theta}^2} + \frac{B}{(1-\hat{\theta})^2}\right)^{-1}\)。

Code 
theta <- seq(0.001, 0.999, .001)
Y <- 30; n <- 100  # 大样本下估计
A <- Y - 0.5; B <- n - Y - 0.5
theta_MAP <- A / (A + B)
Info <- A / theta_MAP^2 + B / (1 - theta_MAP)^2

# 精确后验与高斯近似对比
post1 <- dbinom(Y, n, theta) * dbeta(theta, 0.5, 0.5)
post1 <- post1 / sum(post1)
post2 <- dnorm(theta, theta_MAP, sqrt(1 / Info))
post2 <- post2 / sum(post2)
plot(theta, post1, type = "l", lwd = 2,
     xlab = expression(theta), ylab = "后验密度")
abline(v = theta_MAP, col = 3, lwd = 2)
lines(theta, post2, col = 2, lwd = 2)
legend("topright", c("精确后验", "高斯近似", "MAP"), bty = "n", col = 1:3, lwd = 2)

蒙特卡洛方法

高斯近似用一条正态曲线代替整个后验——快速但丢失了分布的形状细节。蒙特卡洛方法的思路不同：不近似分布形式，而是从（或间接从）后验中抽取大量样本，然后用样本的经验分布逼近真实后验。样本越多，逼近越精确——这由 LLN（Ch03）担保。

逆CDF抽样

如果目标分布的累积分布函数 \(F_X\) 可求逆，抽样极其简单：从 \(U \sim U(0,1)\) 中抽取均匀随机数，取 \(Y = F_X^{-1}(U)\)，则 \(Y \sim F_X\)。为什么成立？

\[F_Y(y) = P(Y \leq y) = P(F_X^{-1}(U) \leq y) = P(U \leq F_X(y)) = F_X(y)\]

反之也成立：对任意连续随机变量 \(X\)，\(F_X(X) \sim U(0,1)\)。这两个事实互为逆命题，构成了逆CDF抽样的全部数学基础。

Code 
# 逆CDF变换验证：从U(0,1)生成指数分布样本
lambda <- 1
u <- runif(1000)
x <- -1 / lambda * log(1 - u)
hist(x, main = "逆CDF抽样：指数分布 Exp(1)", xlab = "x",
     col = "lightblue", border = "white", freq = FALSE)
curve(dexp(x, lambda), add = TRUE, col = "steelblue", lwd = 2)

拒绝抽样

如果 CDF 不可求逆呢？拒绝抽样（Rejection Sampling）提供了替代方案。它的核心思想是：找一个容易抽样的提议分布 \(g(\theta)\)，使放大后的提议分布能完全”包裹”目标分布 \(f(\theta \mid y)\)，然后从 \(g\) 中抽样，按规则拒绝一部分样本，使得保留的样本服从 \(f\)。

符号 含义
\(f(\theta \mid y)\) 目标分布（后验密度）
\(g(\theta)\) 提议分布——容易抽样
\(M = \max_\theta \frac{f(\theta \mid y)}{g(\theta)}\) 包络常数——保证 \(M \cdot g(\theta) \geq f(\theta \mid y)\)
\(\alpha = \frac{f(\theta \mid y)}{M \cdot g(\theta)} \in [0,1]\) 接受概率

算法步骤：

  1. 从 \(g(\theta)\) 中抽取候选样本 \(\theta^*\)。
  2. 从 \(U \sim U(0,1)\) 中抽取随机阈值。
  3. 若 \(U \leq \frac{f(\theta^* \mid y)}{M \cdot g(\theta^*)}\)，接受 \(\theta^*\)；否则拒绝，返回步骤 1。
  4. 重复，直至收集到足够的接受样本。

几何直观：设想两条曲线——上方是放大后的提议分布 \(M \cdot g(\theta)\)，下方是目标分布 \(f(\theta \mid y)\)。从 \(g\) 中抽一个点 \(\theta^*\)（横坐标），再在纵轴上随机选一个高度 \(U \cdot M g(\theta^*)\)。如果这个高度落在下方曲线之下，接受 \(\theta^*\)；落在两条曲线之间，拒绝。正是通过这种随机阈值机制，目标分布密度高的区域接受概率高，密度低的区域接受概率低——最终接受样本的分布严格服从 \(f\)。

接受的样本比例恰好等于 \(1/M\)。因此 \(M\) 越接近 1，抽样效率越高。

如何选择提议分布 \(g(\theta)\)？ 提议分布应具有比目标分布更厚的尾部。例如，若目标为 \(N(0,1)\)，可用 \(t_2\) 分布作为提议——\(t\) 分布的厚尾确保它能覆盖正态分布的远尾区域，避免在极端值处 \(M \cdot g(\theta) < f(\theta \mid y)\)。

curve(dnorm(x), -6, 6, xlab = expression(theta), ylab = "密度", n = 200)
curve(dt(x, 2), -6, 6, add = TRUE, col = 4, n = 200)
legend("topright", c("目标: N(0,1)", "提议: t(2)"), col = c(1, 4), bty = "n", lty = 1)

拒绝抽样的局限：高维空间中，\(M\) 以指数速度膨胀——这是维度灾难的又一次显现（Ch02 的高维几何直觉：高维球体的体积集中在薄球壳上，提议分布极难”包裹”目标分布）。这使得拒绝抽样在高维问题中几乎不可用。这引出了下一种方法：与其拒绝样本，不如给所有样本加权。

重要性抽样

重要性抽样（Importance Sampling）是拒绝抽样的进阶版——用”样本加权”替代”样本拒绝”。核心公式来自一个简单的期望恒等式：

\[E_f[h(\theta) \mid y] = \int h(\theta) f(\theta \mid y) d\theta = \int h(\theta) \frac{f(\theta \mid y)}{g(\theta)} g(\theta) d\theta = E_g\left[\frac{f(\theta \mid y)}{g(\theta)} h(\theta)\right]\]

也就是说：目标分布 \(f\) 下的期望，等于提议分布 \(g\) 下加权版本的期望，权重 \(w_i = f(\theta_i \mid y) / g(\theta_i)\)。

若 \(\theta_1, \ldots, \theta_n \sim g\)，则：

\[\hat{\mu}_n = \frac{1}{n}\sum_{i=1}^n \frac{f(\theta_i \mid y)}{g(\theta_i)} h(\theta_i) = \frac{1}{n}\sum_{i=1}^n w_i \, h(\theta_i) \approx E_f[h(\theta) \mid y]\]

\(w_i\) 称为重要性权重。与拒绝抽样不同，重要性抽样不丢弃任何样本——它只是根据每个样本在目标分布下的相对密度调整其贡献。这使得它在所有样本都被保留的意义上比拒绝抽样更”节俭”。但代价是，如果 \(g\) 与 \(f\) 差异过大，少数样本的权重会主导整个估计，导致有效样本量急剧下降。

马尔科夫链蒙特卡洛（MCMC）

独立蒙特卡洛方法（逆CDF、拒绝、重要性）都依赖一个全局提议分布——需要你事先指定一个覆盖整个参数空间的 \(g(\theta)\)。这在低维可行，在高维是灾难。MCMC 换了一种思路：不尝试一次性从目标分布中独立抽样，而是构造一条马尔可夫链，使其平稳分布恰好为目标后验。链运行足够长之后，链上的状态就近似为来自后验的样本。

什么是马尔可夫链？ 马尔可夫链是一列随机变量 \(\theta^{(1)}, \theta^{(2)}, \ldots\)，满足：对任意 \(t\)，\(\theta^{(t)}\) 在给定所有历史 \(\theta^{(1)}, \ldots, \theta^{(t-1)}\) 下的条件分布，只依赖于最近的值 \(\theta^{(t-1)}\)。换句话说，未来只依赖现在，不依赖过去——这是一种”无记忆”的随机过程。

MCMC 的关键优势：每一步迭代，链的近似分布都在改进（在收敛到目标分布的意义上）。不需要全局提议——只需要在当前值附近做局部跳跃。

Metropolis 算法

Metropolis 算法是 MCMC 的起点。目标是从 \(p(x)\) 中采样。需要一个对称的提议分布 \(q(x^{\text{new}} \mid x^{\text{old}})\)，满足 \(q(x^{\text{new}} \mid x^{\text{old}}) = q(x^{\text{old}} \mid x^{\text{new}})\)——典型的例子是以当前状态为中心的高斯随机游走。

算法步骤：

  1. 在第 \(i+1\) 步，设当前状态 \(x^{\text{old}} = x^{(i)}\)。
  2. 从提议分布生成候选状态 \(x^{\text{new}} \sim q(\cdot \mid x^{\text{old}})\)。计算接受概率： \[\alpha = \min\left(1,\; \frac{p(x^{\text{new}})}{p(x^{\text{old}})}\right)\]
  3. 以概率 \(\alpha\) 接受 \(x^{\text{new}}\)（\(x^{(i+1)} = x^{\text{new}}\)）；以概率 \(1-\alpha\) 拒绝（\(x^{(i+1)} = x^{\text{old}}\)）。

直观理解：如果候选状态的密度高于当前状态（\(p(x^{\text{new}}) > p(x^{\text{old}})\)），\(\alpha = 1\)——总是接受（爬山）。如果候选密度更低，\(\alpha\) 等于密度比——以一定概率”下山”，避免困在局部峰。

Code 
# 目标分布：双峰混合正态
p <- 0.4
mu <- c(-1, 2)
sd <- c(.5, 2)
f <- function(x) {
    p     * dnorm(x, mu[1], sd[1]) +
    (1-p) * dnorm(x, mu[2], sd[2])
}
curve(f(x), -4, 8, n = 301, las = 1, ylab = "密度", xlab = "x")
Code 
# 对称提议分布（随机游走）
q <- function(x) rnorm(1, x, 4)

# 单步转移
step <- function(x, f, q) {
  xp <- q(x)                          # 生成候选状态
  alpha <- min(1, f(xp) / f(x))       # 计算接受概率
  if (runif(1) < alpha) x <- xp       # 按概率接受
  x
}

# 迭代抽样
run <- function(x, f, q, nsteps) {
  res <- matrix(NA, nsteps, length(x))
  for (i in seq_len(nsteps))
    res[i,] <- x <- step(x, f, q)
  return(res)
}

# 执行：初始 x=-10，迭代 1100 次，取后 1000 次（烧脑后）
res <- run(-10, f, q, 1100)
hist(res[101:1100], 50, freq = FALSE, main = "", ylim = c(0, .4), las = 1,
     xlab = "x", ylab = "概率密度")
curve(f(x) / integrate(f, -Inf, Inf)$value, add = TRUE, col = "red", n = 200)

收敛效率：链混合得有多好？

MCMC 样本之间存在自相关——第 \(t\) 个和第 \(t+1\) 个样本不独立。因此样本均值 \(\bar{x}\) 的方差不能简单地用 \(\text{Var}(X)/M\) 计算。考虑自相关后的正确方差为：

\[\text{Var}(\bar{x}) = \frac{\sigma^2}{M} \left(1 + 2\sum_{k=1}^{\infty} \rho_k \right)\]

其中 \(\rho_k\) 为滞后 \(k\) 的自相关系数。定义积分自相关时间（IACT）\(\tau = 1 + 2\sum_{k=1}^{\infty} \rho_k\)，则 \(\text{Var}(\bar{x}) = \sigma^2 \tau / M\)。

ACF（自相关函数）图展示了链在不同滞后距离下的自相关强度。若 ACF 随滞后快速衰减至零，说明链的混合性能好；若缓慢衰减，说明链”粘滞”在局部区域，有效样本量低。

有效样本量（ESS）：\(M_{\text{ESS}} = M / \tau\)。ESS 告诉你：这条长度为 \(M\) 的相关链，其估计精度等价于多少个独立样本。ESS 越高，蒙特卡洛估计越精确。收敛诊断是 MCMC 中最容易被忽视的环节——一条链”看起来收敛了”不等于真的收敛。实践中常使用多链初始化配合 Gelman-Rubin \(\hat{R}\) 诊断：多链的组间方差显著大于组内方差时，说明链尚未收敛。

Metropolis-Hastings 算法

Metropolis 要求提议分布对称——这在很多场景中受限。Metropolis-Hastings（MH）将其推广到任意提议分布。接受概率修正为：

\[\alpha(x^{\text{old}} \to x^{\text{new}}) = \min\left(1,\; \frac{p(x^{\text{new}}) \cdot q(x^{\text{old}} \mid x^{\text{new}})}{p(x^{\text{old}}) \cdot q(x^{\text{new}} \mid x^{\text{old}})}\right)\]

关键点： - 当 \(q\) 对称时（\(q(x' \mid x) = q(x \mid x')\)），修正因子 \(q(x^{\text{old}} \mid x^{\text{new}}) / q(x^{\text{new}} \mid x^{\text{old}}) = 1\)，MH 退化为 Metropolis。 - 修正因子被称为 Hastings 比率——它校正了非对称提议导致的采样偏差：如果从 \(x^{\text{old}}\) 跳到 \(x^{\text{new}}\) 比反方向更容易，接受概率需要打折扣，以补偿这种”方向偏好”。 - MH 保证了马尔可夫链的平稳分布恰好为目标分布 \(p(x)\)——无论提议分布是什么（只要覆盖整个支撑集）。

二维示例：从双峰混合二元正态分布中采样。

Code 
# 构造多元正态密度函数
make.mvn <- function(mean, vcv) {
  logdet <- as.numeric(determinant(vcv, TRUE)$modulus)
  tmp <- length(mean) * log(2 * pi) + logdet
  vcv.i <- solve(vcv)
  function(x) {
    dx <- x - mean
    exp(-(tmp + rowSums((dx %*% vcv.i) * dx)) / 2)
  }
}

Gibbs 采样

Gibbs 采样将 MH 的逻辑推向极致——它把高维采样问题拆解为一系列一维条件分布的采样。每一步只改变参数向量的一个分量，从该分量在给定所有其他分量当前值下的条件分布中抽样。

以二维为例，设目标分布为 \(p(x, y)\)，条件分布 \(p(x \mid y)\) 和 \(p(y \mid x)\) 均可直接采样：

  1. 初始化 \(y^{(0)}\)。
  2. 第 \(t\) 次迭代：从 \(p(x \mid y^{(t-1)})\) 中采样 \(x^{(t)}\)；从 \(p(y \mid x^{(t)})\) 中采样 \(y^{(t)}\)。
  3. 重复，烧脑后样本对 \((x^{(t)}, y^{(t)})\) 近似来自联合分布 \(p(x, y)\)。

Gibbs 是 MH 的特例——其提议分布是目标分布的条件分布本身，因此接受概率恒等于 1：

\[\alpha = \min\left(1,\; \frac{f(x^{(i+1)}, y) \, f(x^{(i)} \mid y)}{f(x^{(i)}, y) \, f(x^{(i+1)} \mid y)}\right) = 1\]

  • 优点：无需调优提议分布，提议总是被接受。
  • 缺点：依赖条件分布的可采样性；当参数高度相关时，每次只沿坐标轴移动，采样极慢（链在参数空间中”爬行”）。
Code 
set.seed(123)
rho <- 0.8; n <- 5000
X <- Y <- numeric(n)
X[1] <- 0; Y[1] <- 0
sd <- sqrt(1 - rho^2)

# Gibbs 核心迭代
for (t in 2:n) {
  X[t] <- rnorm(1, mean = rho * Y[t-1], sd = sd)  # X | Y
  Y[t] <- rnorm(1, mean = rho * X[t],   sd = sd)  # Y | X
}

plot(X, Y, pch = 20, col = rgb(0, 0, 1, 0.4),
     main = "Gibbs采样：二元正态 (ρ = 0.8)",
     xlab = "X", ylab = "Y")

# 叠加真实二元正态等高线

Metropolis within Gibbs

当某个条件分布不易直接采样时，可在该 Gibbs 步骤中用一次 MH 采样替代。这保留了 Gibbs”高维拆低维”的核心优势，同时用 MH 弥补条件分布需可采样的缺陷，适配现实中任意复杂的后验结构。

应用：贝叶斯线性回归

把 MCMC 放在一个具体的统计模型中收束。线性回归模型：

\[y_i = \beta_0 + \beta_1 x_i + \epsilon_i, \quad \epsilon_i \sim N(0, \sigma^2)\]

贝叶斯框架下的后验：

\[p(\beta_0, \beta_1, \sigma^2 \mid y, x) \propto p(y \mid \beta_0, \beta_1, \sigma^2, x) \, p(\beta_0, \beta_1, \sigma^2)\]

给定无信息先验 \(p(\beta, \sigma^2) \propto 1/\sigma^2\)，后验条件分布恰好有解析形式——这使 Gibbs 采样可以直接使用：

  1. \(\beta \mid \sigma^2, y, X \sim N\left(\hat{\beta}_{\text{OLS}},\; \sigma^2 (X'X)^{-1}\right)\)
  2. \(\sigma^2 \mid \beta, y, X \sim \text{Scale-Inv-}\chi^2(n, s^2)\)，其中 \(s^2 = \frac{1}{n}\sum_{i=1}^n (y_i - x_i'\beta)^2\)

Gibbs 迭代交替从这两个条件分布中抽样，收敛后获得 \((\beta, \sigma^2)\) 的联合后验样本——无需解析积分，即可计算回归系数的后验均值、可信区间和预测分布。这正是 Ch10（回归分析）贝叶斯路径的数学地基。

深度思考：

  1. 贝叶斯更新的核心是”今天的后验 = 明天的先验”。如果数据是分批到达的，频率学派需要重新跑整个估计，而贝叶斯只需用新数据更新旧后验。在流式数据场景（实时推荐、金融风控）中，这种”增量学习”的能力意味着什么？

  2. 共轭先验简化了计算，但也限制了先验的选择自由。在什么情况下你愿意放弃共轭的便利，选择一个更能反映真实知识但不共轭的先验？代价是什么？

  3. 贝叶斯因子自动惩罚模型复杂度——这个性质与 AIC/BIC（Ch10）的惩罚项、Lasso（Ch10）的 L1 正则化、以及奥卡姆剃刀原则共享同一个哲学内核：信息不是免费的。 你能找到这三个方法在数学上的共同结构吗？

  4. MCMC 的收敛诊断是贝叶斯计算中最容易被忽视的环节。一条马尔可夫链”看起来收敛了”不等于它真的收敛了——你怎么知道自己没有被困在局部模态中？提示：多链初始化、Gelman-Rubin \(\hat{R}\) 诊断、有效样本量 ESS。

  5. 拒绝抽样的效率由 \(M\) 控制，\(M\) 越接近 1 越好。在高维问题中，\(M\) 会以指数速度膨胀——这是维度灾难在贝叶斯计算中的表现。为什么高维空间中提议分布很难”包裹”目标分布？（提示：Ch02 的高维几何直觉——高维球体的体积集中在薄球壳上。）

  6. Metropolis 算法接受候选状态的概率为 \(\min(1, p(x^{\text{new}})/p(x^{\text{old}}))\)。为什么不是 \(\min(1, p(x^{\text{old}})/p(x^{\text{new}}))\)？如果分子分母颠倒，马尔可夫链的平稳分布会是什么？——提示：考虑细致平衡条件（Detailed Balance）以及为什么 \(\alpha\) 的这一特定形式恰好使 \(p(x)\) 成为平稳分布。
